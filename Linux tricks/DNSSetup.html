ASSISTANT:

Configuring a DNS server with a primary (master) and secondary (slave) setup on Rocky Linux involves installing and configuring BIND (Berkeley Internet Name Domain), which is the most commonly used DNS software. Below is a step-by-step guide to setting up a DNS master-slave configuration.

Prerequisites

  1. Two Rocky Linux servers: Ensure you have two servers. Let's call them dns-master and dns-slave.
  2. Static IP addresses: Assign static IP addresses to both servers.
  3. Root or sudo access: You need administrative privileges to install and configure software.

Step-by-Step Configuration

Step 1: Install BIND on Both Servers

  1. Update the system packages:

    BASH
    sudo dnf update -y

  2. Install BIND on both servers:

    BASH
    sudo dnf install bind bind-utils -y

Step 2: Configure the DNS Master Server

  1. Edit the BIND configuration file:

    Open /etc/named.conf in a text editor:

    BASH
    sudo nano /etc/named.conf

  2. Configure the options block: Ensure the listen-on and allow-query options are correctly set. Set your local network or specific IPs allowed to query.

    BASH
    options {
    listen-on port 53 { 127.0.0.1; your_master_ip; };
    listen-on-v6 port 53 { ::1; };
    directory "/var/named";
    dump-file "/var/named/data/cache_dump.db";
    statistics-file "/var/named/data/named_stats.txt";
    memstatistics-file "/var/named/data/named_mem_stats.txt";
    recursing-file "/var/named/data/named.recursing";
    secroots-file "/var/named/data/named.secroots";
    allow-query { localhost; your_network/24; };
    allow-transfer { your_slave_ip; };
    recursion no;
};

    Replace your_master_ip, your_network/24, and your_slave_ip with the actual IP addresses.

  3. Define the zone: Add a zone block for your domain.

    BASH
    zone "example.com" IN {
    type master;
    file "example.com.db";
    allow-transfer { your_slave_ip; };
};

  4. Create the zone file:

    Create a new zone file at /var/named/example.com.db:

    BASH
    sudo nano /var/named/example.com.db

  5. Add the following zone file information:

    BASH
    $TTL 86400
@   IN  SOA     ns1.example.com. admin.example.com. (
                    2024120501 ; Serial
                    3600       ; Refresh
                    1800       ; Retry
                    604800     ; Expire
                    86400 )    ; Negative Cache TTL
;
@       IN  NS      ns1.example.com.
@       IN  NS      ns2.example.com.
ns1     IN  A       your_master_ip
ns2     IN  A       your_slave_ip

    Replace your_master_ip and your_slave_ip with the actual IPs.

  6. Change permissions:

    BASH
    sudo chown root:named /var/named/example.com.db
sudo chmod 640 /var/named/example.com.db

  7. Start and enable BIND:

    BASH
    sudo systemctl enable named
sudo systemctl start named

  8. Check the BIND configuration:

    BASH
    sudo named-checkconf
sudo named-checkzone example.com /var/named/example.com.db

Step 3: Configure the DNS Slave Server

  1. Edit the BIND configuration file on the slave server:

    BASH
    sudo nano /etc/named.conf

  2. Configure the options block: Allow queries from your network.

    BASH
    options {
    listen-on port 53 { 127.0.0.1; your_slave_ip; };
    listen-on-v6 port 53 { ::1; };
    directory "/var/named";
    dump-file "/var/named/data/cache_dump.db";
    statistics-file "/var/named/data/named_stats.txt";
    memstatistics-file "/var/named/data/named_mem_stats.txt";
    recursing-file "/var/named/data/named.recursing";
    secroots-file "/var/named/data/named.secroots";
    allow-query { localhost; your_network/24; };
    recursion no;
};

  3. Define the slave zone:

    BASH
    zone "example.com" IN {
    type slave;
    file "slaves/example.com.db";
    masters { your_master_ip; };
};

  4. Create the directory for slave zones:

    BASH
    sudo mkdir -p /var/named/slaves

  5. Change permissions:

    BASH
    sudo chown named:named /var/named/slaves

  6. Start and enable BIND:

    BASH
    sudo systemctl enable named
sudo systemctl start named

  7. Check the BIND configuration:

    BASH
    sudo named-checkconf

Step 4: Firewall Configuration

Ensure that port 53 is open on both servers.

BASH
sudo firewall-cmd --permanent --add-port=53/tcp
sudo firewall-cmd --permanent --add-port=53/udp
sudo firewall-cmd --reload

Step 5: Testing

  1. Test from the master server:

    BASH
    dig @localhost example.com

  2. Test from the slave server:

    BASH
    dig @localhost example.com

  3. Test from a client machine within your network to ensure it can resolve queries from both the master and the slave.

This setup provides a basic DNS master-slave configuration. For production environments, consider securing your DNS servers further with DNSSEC, implementing monitoring, and setting up logging.
